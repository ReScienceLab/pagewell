# Comparison Page Rules

Alternative and comparison pages are high-intent but high-risk.

## Required posture

Be factual, fair, and useful.

Use:

- "When to choose X"
- "When to choose Y"
- "Designed for"
- "May be a fit if"

Avoid:

- unsupported "best" claims
- competitor FUD
- invented feature gaps
- made-up pricing or customer data

## Minimum structure

1. Hero
2. Decision summary
3. What each product/alternative is
4. Factual comparison table
5. When to choose each
6. FAQ
7. CTA

## Always run

- `skills/qa.md`, including the claim-check subroutine
