# Skill: Brainstorm Free Tools

## Purpose

Identify small, useful, no-signup tools that a product's ICP already searches for.

Free tools are a Sitewright page type and acquisition strategy: give immediate utility first, earn trust, then route qualified users toward the main product.

## Philosophy

A good free tool is not a thin landing page. It is a working utility that:

- solves one narrow problem immediately
- requires no signup before first value
- targets a real search query
- is easy to share and bookmark
- naturally connects to the main product
- is cheap to build and maintain

Think: generators, converters, analyzers, validators, calculators, and previewers.

## Required inputs

- `SITE_PROFILE.md`
- `DESIGN.md`
- canonical product/company facts
- target product or business area
- ICP details if available
- optional keyword data or competitor examples

## Brainstorming process

Generate at least 15 free tool ideas across these categories:

- **Generators**: templates, titles, prompts, READMEs, snippets, boilerplates
- **Converters**: format A to format B
- **Analyzers/checkers**: validators, linters, graders, audits
- **Calculators**: ROI, pricing, sizing, estimates, benchmarks
- **Previewers**: live renderers, markdown previews, schema previews, OG previews

For each idea, provide:

1. Tool name
2. One-sentence utility
3. Target search query
4. Main input
5. Main output
6. How it connects to the product
7. Build complexity: Low / Medium / High
8. Maintenance risk: Low / Medium / High

## Scoring matrix

Score each idea from 1-5:

| Criterion | Meaning |
|---|---|
| Search intent | Would the ICP search for this exact utility? |
| Build speed | Can v1 be built quickly and safely? |
| Product fit | Does it naturally lead to the main product? |
| Uniqueness | Is the SERP/opportunity not completely saturated? |
| Utility | Does the tool produce real value without signup? |

Output a table:

| Tool | Search Intent | Build Speed | Product Fit | Uniqueness | Utility | Total |
|---|---:|---:|---:|---:|---:|---:|

## Selection rules

Pick the top 3 ideas, but only if they meet all of these conditions:

- v1 can be implemented as one focused page or small route group
- no authentication required
- no expensive backend dependency unless approved
- no legal/security/compliance risk
- product CTA is natural, not forced

If none qualify, say so and ask for a narrower ICP or product angle.

## Output

Create a recommendation report:

```md

...

```

## Acceptance criteria

- At least 15 ideas considered.
- Top 3 are scored and justified.
- Recommendations prioritize usefulness and search intent over novelty.
- No tool requires signup before first value.
